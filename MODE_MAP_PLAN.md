# 模式地图规划

## 当前结论

- 地图开发可以先共用 `ranked-core-v1` 这套竞技底盘，但不能继续让所有模式都长得像排位赛。
- 真正要复用的是底层脚手架：出生逻辑、基础资源刷新、扎刺基础规则、中心争夺判定。
- 真正要拉开的是模式体验：地图结构、危险机制、资源热点、路线选择、终局节奏。

## 共用底盘

所有模式的地图先按同一套生产模板拆分：

1. 出生区
2. 中心争夺区
3. 侧路发育区
4. 回旋/脱战区
5. 模式专属危险层
6. 模式专属资源层

这样后面做新图时，不会出现“只是换皮，没有玩法差异”的问题。

## 四个模式的地图方向

### 排位赛

- 定位：公平竞技底盘。
- 核心关键词：对称、稳定、可控。
- 地图结构：镜像出生点 + 中路高价值控制区 + 双边发育廊。
- 重点不是花哨机制，而是强度平衡和清晰的中路博弈。
- 必须保证任意出生侧都不能天然优势。

### 巅峰赛

- 定位：排位赛的高压精英版。
- 核心关键词：更收、更冷、更少容错。
- 地图结构：沿用排位底盘，但压缩主战区、减少绕路空间、加强终局收口。
- 巅峰赛不应该只是“排位换紫色主题”，而是要让强碰撞更早发生、失误代价更大。
- 适合加入固定窄口、刺阵门、焦点终局桥之类的高压结构。

### 经典模式

- 定位：最顺手、最熟悉、最适合练球感。
- 核心关键词：开放、平滑、轻快。
- 地图结构：开放式成长圈 + 平滑资源环 + 可读性强的绕路线。
- 经典模式要减少强制高压机制，让玩家感受到“顺滑发育”和“熟悉感”。
- 刺球可以保留，但密度和惩罚应低于排位、巅峰。

### 大逃杀

- 定位：高压生存图，必须和竞技图彻底区分开。
- 核心关键词：危险、迁移、路线决策、生存感。
- 地图结构：大尺度生存地图 + 安全圈迁移 + 外环补给点 + 终圈高光区。
- 必做机制：
  - 安全圈和圈外持续伤害
  - 红区
  - 吊刺区
  - 护盾哨站
  - 终圈盆地
- 大逃杀不能只是“排位图 + 缩圈”，而是要让玩家持续面临是否进圈、是否绕红区、是否抢护盾、是否卡终圈的决策。

## 推荐开发顺序

1. 先完成排位赛标准地图，作为所有模式的基础模板。
2. 基于同一底盘分出经典模式，优先做开放感和成长感。
3. 再做巅峰赛，把排位图收紧成高压精英版。
4. 最后做大逃杀专属机制层，把红区、吊刺、护盾、安全圈全部叠起来。

## 设计约束

- 每张图至少要有一条主动进攻路线。
- 每张图至少要有一条保守运营路线。
- 每张图都要有明确的视觉锚点，让玩家截图就能认出模式。
- 每个模式前 90 秒的核心决策必须不同。
- 不允许用同一张图只改文案就算“新模式地图”。

## 验收标准

- 只看小地图或俯视截图，也能分辨出是排位、巅峰、经典还是大逃杀。
- 玩家第一次进入该模式的 30 秒内，就能感觉到这张图的节奏差异。
- 大逃杀必须明显比其他三个模式更危险。
- 巅峰赛必须明显比排位更高压。
- 经典模式必须明显比排位更顺滑、更自由。
